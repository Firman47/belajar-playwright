# Bug Report

**ID Test:** REG-017, REG-019, REG-022, REG-025

**Judul:** Toast error selalu menampilkan "Registration failed" — tidak menampilkan pesan spesifik dari API

**Klasifikasi:** BUG_APP

**Deskripsi:**
Ketika API mengembalikan validation error spesifik (seperti "email must be an email", "Username must be 3-64 characters..."), toast error di UI selalu menampilkan pesan generik "Registration failed" — sama persis di title dan description. Tidak ada feedback yang informatif ke user.

**Test Cases Terkait:**
| ID | Input | API Response | API Message | UI Toast |
|----|-------|-------------|-------------|----------|
| REG-017 | Username 300 chars | 400 | "Validation failed" + nested constraint "username must be 3-64 characters..." | "Registration failed" |
| REG-019 | Email 300 chars | 400 | "Validation failed" + "email must be an email" | "Registration failed" |
| REG-022 | Username @#$% | 400 | "Validation failed" + "username can only contain letters, numbers, dots, underscores, and hyphens" | "Registration failed" |
| REG-025 | Username with spaces | 400 | "Validation failed" + "username can only contain letters, numbers, dots, underscores, and hyphens" | "Registration failed" |

**Langkah Reproduksi (contoh REG-017):**

1. Buka halaman Register
2. Isi semua field dengan data valid, kecuali username = "a".repeat(300)
3. Klik Register
4. Lihat toast error

**Expected:**
Toast description menampilkan pesan error spesifik dari API, misal "Username must be 3-64 characters and can only contain letters, numbers, dots, underscores, and hyphens"

**Actual:**
Toast description hanya menampilkan "Registration failed" — tidak informatif

**Response API (REG-017):**

```json
{
  "status": false,
  "message": "Validation failed",
  "data": [
    {
      "constraints": {
        "matches": "username must be 3-64 characters and can only contain letters, numbers, dots, underscores, and hyphens"
      }
    }
  ]
}
```

**Root Cause:**
Toast component di frontend tidak mem-parsing dan menampilkan validation errors dari nested `data[].constraints` yang dikembalikan API. Selalu fallback ke pesan generik.

**Evidence:**
- Playwright HTML Report (trace & screenshot tersedia)
- Semua REG-003, 007, 017, 019, 022, 025 menunjukkan pola yang sama

**Status:** Open
